# Repository Guidelines

## Project Structure & Module Organization
`src/` contains the TypeScript source for this repository; `src/index.ts` is the current entrypoint. `dist/` is the compiler output directory generated by TypeScript and should be treated as build artifact output, not hand-edited source. Root configuration lives in `package.json` and `tsconfig.json`. Local-only paths already ignored include `node_modules/`, `/tmp`, `/out-tsc`, and `.vscode/`.

## Build, Test, and Development Commands
- `npm install`: install the local TypeScript toolchain.
- `npm run build`: compile `src/` into `dist/` using the strict compiler settings in `tsconfig.json`.
- `npx tsc --watch`: optional watch mode for local iteration when changing TypeScript files.

There is no test script yet. If you introduce tests, add the command to `package.json` in the same change so contributors have a single entrypoint.

## Coding Style & Naming Conventions
Write code for strict TypeScript: avoid `any`, type public interfaces explicitly, and keep modules small and focused. Prefer 2-space indentation, `camelCase` for variables and functions, and `PascalCase` for classes or types. Use lowercase or kebab-case filenames for new modules, for example `src/arg-parser.ts`. Keep reusable logic in dedicated modules under `src/` and leave top-level side effects in the entrypoint only.

## Testing Guidelines
No test framework is configured yet, so every behavior change should at minimum pass `npm run build`. When adding tests, prefer `*.test.ts` naming and either colocate them with the source file or place them under `src/__tests__/`. Keep tests small and focused on observable behavior.

## Commit & Pull Request Guidelines
This repository has no commit history yet, so there is no established convention to copy. Start with short, imperative commit messages; Conventional Commit style is a good default, for example `feat: add CLI argument parsing` or `fix: handle empty input`. Pull requests should explain the change, link related issues, list verification steps, and include sample CLI output when behavior changes.

## Configuration Notes
The compiler targets CommonJS on `es2016` with `strict` mode enabled. Do not commit editor-specific files, temporary artifacts, or generated output unless the release process explicitly requires it.
